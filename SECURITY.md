# Security Policy

## Supported versions

The latest state of `main` is the supported version.

## Reporting a vulnerability

Please report security issues **privately** via GitHub's **[Report a vulnerability](https://github.com/StChiotis/Library-First-Engineering/security/advisories/new)** (the *Security* tab → *Advisories*) rather than opening a public issue. You can expect an acknowledgement within a few days.

## Scope

Library-First Engineering ships no executable code — the framework is protocol documents, persona contracts, and skill prompts (prose). Reports are still welcome for:

- **Instruction-surface issues**: anything in the protocol or skill prompts that could be abused to steer an agent into unsafe behavior (e.g. prompt-injection vectors in files an agent is instructed to read and obey).
- **Repository-surface issues**: malicious or compromised links, templates, or metadata.

Ordinary protocol bugs and documentation drift are handled as public [Issues](https://github.com/StChiotis/Library-First-Engineering/issues).
